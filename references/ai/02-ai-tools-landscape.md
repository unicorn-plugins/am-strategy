# 02 AI 도구 랜드스케이프와 트렌드

## 1. 왜 지금, AI 컨설팅의 변곡점인가

2026년은 "기술 선택"보다 "거버넌스 + 아키텍처 설계"가 컨설팅 가치를 결정함.

### 1.1 변화의 축 (2024년 이전 → 2025~2026년)

| 변화의 축 | 2024년 이전 | 2025~2026년 |
|-----------|-------------|--------------|
| 경쟁 축 | 모델 성능 경쟁 | 에이전트 플랫폼·거버넌스 경쟁 |
| 비용 구조 | 토큰 단가 중심 | 사고 시간(thinking time) 변수 추가 |
| 연결 표준 | 벤더별 SDK N×M | MCP 표준(USB-C) 1종 |
| 규제 환경 | 가이드라인 수준 | 법적 의무 (한국 AI 기본법 2026.1.22, EU AI Act) |

### 1.2 주요 시장 시그널
- 토큰 가격 1,000배 ↓ + 멀티에이전트 워크플로우 327% ↑ (2025년 6~10월)  
  → 모델 자체가 차별화 요소가 되기 어려움. 가치의 축이 모델 → 에이전트·거버넌스로 이동.  
- 엔터프라이즈 LLM 1위 교체: Claude 40% > OpenAI 27% (Menlo Ventures)  
  → 단일 벤더 락인 위험, 멀티모델 라우팅이 표준 전략.  

### 1.3 컨설턴트 질문의 전환
"어떤 모델을 고를 것인가" → "멀티 레이어(모델·코딩·워크플로우·에이전트·클라우드)를 어떻게 엮을 것인가."

> 출처: Andreessen Horowitz, databricks.com, Menlo (2025.12)

## 3. 5개 레이어 프레임워크 — AI 도구 스택의 전체 지형

AI 도구는 독립 제품이 아니라 5개 레이어의 통합 스택으로 의사결정해야 함. 레이어별이 아닌 일관된 전략으로 통합 필요.

| 레이어 | 명칭 | 핵심 트렌드 | 대표 제품 |
|--------|------|-------------|-----------|
| 1 | 파운데이션 모델 (Foundation Model) | "생각하는 모델"이 새 표준, 가격 1,000배 붕괴 | Claude · GPT · Gemini · DeepSeek |
| 2 | AI 코딩 도구 (AI Coding Tools) | 개발자 85% 사용, 신규 코드 46% AI 생성, $85억 시장 | Claude Code · GitHub Copilot · Cursor |
| 3 | 워크플로우 빌더 (Workflow Builders) | 비개발자 자동화, 노드 기반, AI 네이티브로 진화 | n8n · Make · Zapier · Dify |
| 4 | 에이전트 프레임워크 (Agent Frameworks) | 멀티에이전트 +327% 성장, MCP 표준으로 통합 연결 | LangGraph · CrewAI · Microsoft Agent Framework |
| 5 | 클라우드 AI 플랫폼 (Cloud AI Platforms) | 3강 구도(AWS·MS·GCP) + 소버린 AI 가속 | AWS Bedrock · Azure OpenAI · GCP Vertex AI |

### 3.1 통합 원칙
1. **일관된 전략으로 통합**: 레이어별 Best-of-breed ≠ 최적. 레이어 간 호환·거버넌스 고려 필수.  
2. **거버넌스-퍼스트 순서**: (0) 거버넌스 → (1) 모델 → (2)~(4) 실행 도구 → (5) 인프라.  
3. **컨설턴트 체크포인트**: 고객사가 각 레이어에 "확정된 솔루션"을 갖고 있는가? — 공백이 곧 기회.

## 6. Layer 3 — 워크플로우 빌더

### 6.1 AI 네이티브 자동화의 부상

"if-then 자동화"에서 "LLM이 자율 의사결정하는 AI 네이티브 자동화"로 전환. 2032년 $455억 시장 (Verified Market Research).

| 시점 | 방식 | 특징 |
|------|------|------|
| BEFORE | Rule-based iPaaS/RPA | if-then 고정 로직, 수작업 예외 처리 |
| AFTER (2026~) | AI 네이티브 자동화 | LLM이 맥락 읽고, 자율 의사결정, 예외까지 흡수 |

### 6.2 주요 제품

| 제품 | 대상·라이선스 | 핵심 특징 |
|------|----------------|-----------|
| **n8n** | 개발자·셀프호스팅 (매출 10배·$25억) | AI 네이티브 자동화 대표 제품, 워크플로우 80% AI 에이전트 내장, 워크플로우 단위 과금 |
| **Zapier** | 비개발자·SaaS (8,000+ 앱) | 업무자동화의 전통적 강자, MCP 지원 |
| **Make** | 반기술자·SaaS (Zapier 대비 1/3 비용) | Zapier의 가성비 대안, AI Agents 베타(2025.10) |
| **Dify** | 개발자·오픈소스 (GitHub ★10만) | RAG 내장(하이브리드+리랭킹), 워크플로우를 API와 MCP로 노출 가능 |
| **Flowise** | LLM 개발자·오픈소스 (Node.js 기반) | Dify의 경량 대안, 고급 파이프라인 설계, Agentflow 멀티에이전트 |
| **Langflow** | 연구자·개발자·오픈소스 (MCP 네이티브) | LangGraph의 공식 Visual Builder, Python 기반, 워크플로우를 MCP 서버로 노출 가능 |
| **Coze** | 비개발자·오픈소스 (ByteDance·Coze 2.0) | 중국 대표적 워크플로우 빌더, Skills Marketplace, Agent Plan(장기 자율 실행) |

> RAG (검색 증강 생성, Retrieval-Augmented Generation): LLM이 응답 시 외부 데이터(문서·DB·API)를 검색·활용하는 패턴. 파인튜닝 없이 최신 정보·사내 지식 활용, 환각 방지, 출처 추적성 확보 → 엔터프라이즈 LLM 도입의 사실상 표준.

### 6.3 선택 의사결정 트리

### 7.4 MCP vs A2A

#### 채택 지표
- 9,700만+ 월 SDK 다운로드 (MCP 채택 폭발적 확산)  
- 5,800+ MCP 서버 (공개), 12개월 만에 표준화  
- 300+ 클라이언트 (Claude · ChatGPT · Cursor · Gemini · VS Code)  

#### 프로토콜 비교

| 항목 | MCP (Model Context Protocol) | A2A (Agent-to-Agent) |
|------|------------------------------|----------------------|
| 역할 | "도구 연결" 수직 (LLM ↔ 도구) | "에이전트 간 협업" 수평 (에이전트 ↔ 에이전트) |
| 주요 특징 | Anthropic 2024.11 공개, LLM ↔ 도구 표준, Linux Foundation 기부, 12개월 만에 사실상 표준 | Google 2025.4 발표, HTTP / JSON-RPC, "Agent Card" 발견 메커니즘 |
| 채택도 | 월 SDK 9,700만+ · 서버 5,800+ · 클라이언트 300+ (Claude · ChatGPT · Cursor · Gemini · VS Code · Notion · Stripe) | 150+ 조직 지원 (Adobe · ServiceNow · SAP 등). MCP 대비 실제 채택은 느림 |

> N×M → N+M 표준화 효과: MCP는 AI 도구 연결의 커넥터 폭발을 막음 → 자사 시스템을 MCP 서버로 노출한 기업이 에이전트 시대 경쟁 우위 확보.

#### 한국 특화 MCP 서버
글로벌 AI 도구의 한국 정부·기업 시스템 연동 격차를 메움.

| 시스템 | 기능 |
|--------|------|
| DART | 전자공시 |
| HWP 문서 | 한글 파일 변환 |
| 국가법령정보 | 법령·판례 검색 |
| KIPRIS | 특허 검색 |
| 공공데이터포털 | OpenAPI 연동 |

> 컨설팅 노트: 내부 API MCP化 로드맵 → 고객사 차별화 자산. 조기 투자 → 선발주자가 생태계 주도. 한국 특화 MCP → 국내 기업만의 차별화 영역.

### 9.2 추론 경제학의 새 국면
- 3B 소형 모델이 고급 추론으로 70B 대형 모델 능가 가능  
- "사고 시간(thinking time)"이 비용 변수 → 추론 강도 조절 전략 필요 (low / medium / high)  

> Note: 4가지 패턴은 상호배타가 아니라 조합 가능 (예: 최상위 Supervisor + 하위 Role-based Crew). 고객사의 워크플로우 복잡도에 맞춰 추천.

## 11. [참고] 한국 AI 기본법 — 거버넌스 퍼스트의 법적 근거

핵심: 2026년 1월 22일부터 AI 도입은 "기술 선택 전에 거버넌스 수립"이 법적 의무.

### 11.1 핵심 조항

| 영역 | 의무 내용 |
|------|-----------|
| 투명성 | 생성형 AI 결과물 명시 의무 |
| 안정성 | 위험 사전 파악, 모니터링 체계, 사고 대응 절차 수립 |
| 고영향 AI 특별 책무 | 의료기기, 에너지/수도, 교통, 채용, 대출심사, 공공서비스, 학생평가 7개 분야 사전 관리 |
| AI 영향평가 | EU AI Act 유사 사전 위험 평가 (규제보다 진흥에 무게) |
| 국내 대리인 지정 | 해외 빅테크의 국내 대리인 지정 의무화 |

### 11.2 시행 일정
2026년 1월 22일 정식 시행. 1년 이상 계도 기간 후, 빠르면 2027년 이후 과태료 부과.

### 11.3 결합 규제 환경
- EU AI Act (2025.8 발효): 최대 €3,500만 또는 글로벌 매출 7% 과징금  
- 역외 적용: 한국 시장에 영향을 미치는 해외 AI에도 적용 → 글로벌 AI 도구 벤더도 규제 대상  

### 11.4 컨설턴트 체크리스트
1. **고영향 AI 목록**: 고객사에 고영향 AI 목록이 작성되어 있는가?  
2. **리스크 평가 프로세스**: 리스크 평가 프로세스가 수립되어 있는가?  
3. **감사 로그**: 모든 AI 호출·응답의 감사 로그 수집·보존·질의가 가능한가?  
4. **에이전트 접근 통제**: MCP Gateway 또는 제로 트러스트로 에이전트 도구 접근이 통제되는가?  

> Tip: 2026년 컨설팅 제안의 'Page 1'은 반드시 거버넌스로 시작. 1년 계도 기간은 의무가 없다는 뜻이 아니라 지금이 준비 기간이라는 뜻.
