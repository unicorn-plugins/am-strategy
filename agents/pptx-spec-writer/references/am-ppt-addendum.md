# AM 전략 PPT 추가 가이드 (am-ppt-addendum)

> **상속**: 본 addendum은 `{DMAP_PLUGIN_DIR}/resources/guides/office/pptx-build-guide.md`를 **상위 표준**으로 함.
> 컬러·타이포·레이아웃 패턴(A~F)·컴포넌트·디자인 규칙·코드 검증 규칙(6절 11항)은 DMAP 가이드를 따름.

## 1. 경영진 소통 순서 (MUST)

모든 경영진 발표본은 다음 **5 스테이지 순서**를 기본 골격으로 채택:

| # | 스테이지 | 전달 질문 | 핵심 콘텐츠 | 권장 장수 |
|---|---------|----------|------------|----------|
| 1 | **왜 (WHY)** | 왜 지금 AM인가? | 5 동인·L1 KPI·경영진 스폰서십 | 3~5 |
| 2 | **얼마 (TCO/BEP)** | 얼마나 드는가? 언제 회수되는가? | 3 시나리오 TCO·BEP·누적 곡선 | 2~3 |
| 3 | **언제 (Phase)** | 언제 무엇을 하는가? | Phase 로드맵·게이트·마일스톤 | 2~3 |
| 4 | **어떻게 (HOW)** | 어떻게 전환하는가? | 6R 매핑·포트폴리오·Bounded Context | 5~8 |
| 5 | **위험 (RISK)** | 무엇이 어긋날 수 있는가? | 리스크 히트맵·가드레일·거버넌스 | 2~3 |

**원칙**: 현황 진단·변화관리·부록은 위 5 스테이지를 보강하는 형태로 배치. 현황 진단을 맨 앞에 두지 말 것(경영진은 "왜 지금"을 가장 먼저 궁금해함).

DMAP 가이드 패턴(A~F)에 AM 도메인 콘텐츠를 매핑:

| 콘텐츠 유형 | 권장 패턴 | 시각 요소 | 근거 |
|------------|----------|----------|------|

| L1 KPI 대시보드 | F (멀티 섹션) | KPI 카드 4~6개 + 스파크라인 | 한눈에 조망 |
| 3 시나리오 TCO | D (테이블+배지) | Conservative/Base/Optimistic 3열 표 + 다크 배지 | 수치 비교 |

| Phase 로드맵 간트 | C (플로우+코드) 변형 | Phase 1~N 세로 바 + 게이트 체크 표 | 단계 흐름 |
| 6R 포트폴리오 매트릭스 | B + 이미지 | 2D 매트릭스 PNG + 범례 | Retain/Rehost/Replatform/Refactor/Rebuild/Replace |
| Bounded Context 맵 | B + 이미지 | 컨텍스트 맵 PNG | DDD 시각화 |
| 12 차원 스코어카드 | D | 12행 표 + 등급 색상 셀 | 조직·기술 4×4+8 차원 |
| A/B/C 등급 요약 | E (카드 그리드) | 3 카드 (A/B/C 각각) | 분류 정리 |
| 리스크 히트맵 | B + 이미지 | 5×5 확률×영향 매트릭스 | 리스크 시각화 |
| AI 정책 3-bucket | E (2×3 카드) | 허용/조건부/금지 카테고리 | 정책 분류 |
| 거버넌스 조직도 | B | 조직 구조 PNG + 우측 R&R 설명 | 관계 구조 |
| 변화관리 단계 | C (플로우) | 인식→실행→정착 세로 플로우 | 단계적 흐름 |

## 3. 수치 표기 규칙 (MUST)

모든 예산·기간·효과 수치는 **3 시나리오 범위**로 표기. 단일 숫자(거짓 정확도) 금지:

| ✅ 권장 | ❌ 금지 |
|--------|--------|
| `예산: 12~18억 (Base 15억)` | `예산: 15억` |
| `Phase 1: 6~9개월` | `Phase 1: 7개월` |
| `효과: 운영비 20~30% 절감` | `효과: 25% 절감` |

**예외**: 룰북 규칙 ID, 게이트 통과 기준(예: CPU<40%), 실측값(예: 현재 MTTR 6시간)은 단일 숫자 허용.

## 5. 강조 배지 용어집 (AM 도메인)

DMAP 가이드의 강조 배지(녹색 `#059669`) 텍스트 표준:

| 배지 | 용도 |
|------|------|
| `WHY` | WHY 정의 섹션 표지 |
| `Strangler Fig` | 점진적 전환 패턴 강조 |
| `Bounded Context` | DDD 경계 슬라이드 |
| `Phase N` | 로드맵 Phase 표지 |
| `Gate` | 게이트 통과 조건 슬라이드 |
| `6R` | Retain/Rehost 등 6R 슬라이드 |
| `TCO` | 재무 분석 슬라이드 |
| `Risk Top N` | 리스크 우선순위 |

다크 배지(`#404155`)는 카테고리명 — `Guardrail`, `AI Policy`, `ARB`, `DevSecOps` 등에 사용.

## 7. Executive Summary 1 슬라이드 규칙

표지 직후 **반드시 1장**의 Executive Summary를 배치:

- 패턴: F (멀티 섹션 대시보드)
- 구성: 상단 1줄 헤드라인 + 4 섹션 (WHY 1줄·TCO 1줄·Phase 1줄·Top Risk 1줄)
- 강조 배지: 헤드라인 좌측 `WHY` 배지
- 이 슬라이드만 보면 의사결정이 가능해야 함 (Elevator Pitch 원칙)

---

## 8. 최소/최대 장수 규칙

- **최소**: 20장 (표지·Exec Summary·5 스테이지·결론 포함)
- **최대**: 35장 (부록 포함). 초과 시 addendum 섹션을 축약하거나 부록 분리.
- 요약본과 별도로 경영진 발표본은 **발표 35~45분** 기준으로 설계 (슬라이드당 90~120초).
