## 1. DORA 5개 핵심 지표 — HBT 현재 위치 및 목표 범위

### 1.1 Deployment Frequency (배포 빈도)

| 구분 | 값 | DORA Top % 위치 | 근거 |
|------|----|----------------|------|
| **현재 (HBT)** | 월 1회 | 상위 **76.1%** (중하위권) | HBT 내부 2025 DevOps 성숙도 진단 |
| **1년차 목표 (범위)** | 주 1~2회 | 상위 **44.6% ~ 76.1%** 구간 진입 | DORA 2025: 주 1회 이상 = 상위 44.6% |
| **3년차 목표 (범위)** | 일 1회 이상 ~ 주 2~3회 | 상위 **22.7% ~ 44.6%** 구간 진입 | DORA 2025: 일 1회 이상 = 상위 22.7% |
| **출처** | Google DORA 2025, 2025.10, https://dora.dev/research/2025/dora-report/ ; HBT 2025 DevOps 진단 | | |

**시사점**: 월 1회 → 주 1회는 "상위 76.1% → 44.6%"로 약 30%p 도약.  
경영진 어젠다와 정확히 일치하는 1차 목표.

### 1.2 Lead Time for Changes (리드타임)

| 구분 | 값 | DORA Top % 위치 | 근거 |
|------|----|----------------|------|
| **현재 (HBT)** | 평균 **22주** (약 5개월) | 상위 **84.7% 외부 (하위권)** | HBT 2025 IT 성과 리포트 |
| **1년차 목표 (범위)** | 1주 ~ 1개월 | 상위 **56.4% ~ 84.7%** 구간 진입 | DORA 2025: 1주 이내 = 상위 56.4% |
| **3년차 목표 (범위)** | 1일 ~ 1주 | 상위 **24.4% ~ 56.4%** 구간 진입 | DORA 2025: 1일 이내 = 상위 24.4% |
| **출처** | Google DORA 2025, 2025.10 ; HBT 신상품 출시 리드타임 22주 (company-profile §2.1) | | |

**시사점**: 현재 하위권에서 **1년차 중위권 + 3년차 상위권** 진입.  
경영진 목표 "22주 → 10주"는 상위 56.4% 진입에 해당 (현실적 1차 목표).

### 1.3 Failed Deployment Recovery Time (복구 시간)

| 구분 | 값 | DORA Top % 위치 | 근거 |
|------|----|----------------|------|
| **현재 (HBT)** | 평균 **4시간** | 상위 **21.3% ~ 56.6%** 구간 (중위권) | HBT NOC 2025 운영 리포트 |
| **1년차 목표 (범위)** | 1~4시간 (안정화) | 상위 **21.3% ~ 56.6%** 유지 + 분산 축소 | DORA 2025: 1일 이내 = 상위 56.6% |
| **3년차 목표 (범위)** | 30분 ~ 1시간 | 상위 **21.3% 이내** 진입 | DORA 2025: 1시간 이내 = 상위 21.3% |
| **출처** | Google DORA 2025, 2025.10 ; 행정안전부 2026.03.11 (장애 81% 감소) ; HBT NOC 리포트 | | |

행안부 사례: 전환 후 장애 **81% 감소** 검증 → HBT 민간 보수 적용 시 50~70% 감소 현실적.

### 1.4 Change Fail Rate (CFR)

| 구분 | 값 | DORA Top % 위치 | 근거 |
|------|----|----------------|------|
| **현재 (HBT)** | **22%** | 상위 **62.2% ~ 81.6%** 구간 (중하위권) | HBT 2025 배포 실패율 리포트 |
| **1년차 목표 (범위)** | 10% ~ 16% | 상위 **36.2% ~ 62.2%** 구간 진입 | DORA 2025: 8~16% = 상위 62.2% |
| **3년차 목표 (범위)** | 4% ~ 10% | 상위 **16.7% ~ 36.2%** 구간 진입 | DORA 2025: 4~8% = 상위 36.2% |
| **출처** | Google DORA 2025, 2025.10 ; HBT 배포 실패율 22% (company-profile §2.1) | | |

**시사점**: CFR 감소는 **테스트 자동화 + 블루/그린 배포 + Canary** 복합 적용 시 달성.  
경영진 보고 시 "배포 100회 중 실패 22건 → 10건 이하"로 **체감적 전환** 메시지화 가능.

## 2. TCO 절감 정량화 — 3 시나리오 (Conservative / Base / Optimistic)

### 2.1 시나리오 설계

한국 공공부문 **검증된 7년 누적 18.4% 절감**을 기준점으로, 민간 통신업의 특성(변경 빈도↑·B2C↑·내부 인력 비중↑)을 반영하여 3가지 시나리오 제시.

| 시나리오 | 7년 누적 절감률 | 3년 누적 절감률 (선형 가정) | 근거 |
|--------|-------------|----------------------|------|
| **Conservative** | 12% ~ 15% | 5% ~ 7% | 공공 18.4% × 민간 보수 보정 (0.65~0.80배) |
| **Base** | 15% ~ 20% | 7% ~ 10% | 공공 18.4% 직접 적용 + DORA 주 1회 배포 진입 효과 |
| **Optimistic** | 20% ~ 30% | 10% ~ 15% | DORA Cluster 6·7 (상위 40%) 진입 + AI ROI 잠금 해제 |

### 2.2 경영진 제시 목표와 매핑

**경영진 제시: 3년 내 IT 운영비 15% 절감 (6,200억 → 5,270억, 누적 약 2,790억)**

| 시나리오 | 3년 누적 절감 금액 | 경영진 목표 달성도 |
|--------|----------------|---------------|
| Conservative (5~7%) | 약 930억 ~ 1,300억 | **33% ~ 47%** — 목표 미달 (보완 전략 필수) |
| Base (7~10%) | 약 1,300억 ~ 1,860억 | **47% ~ 67%** — 목표 근접 |
| **Optimistic (10~15%)** | 약 **1,860억 ~ 2,790억** | **67% ~ 100%** — **목표 달성 시나리오** |

> **지민 코멘트**: 경영진 "15% 절감"은 **Optimistic 시나리오** 의 상단에 해당.  
> DORA Cluster 6·7 진입 + AI ROI 잠금 해제가 동시 달성되어야 가능.  
> **Conservative 시나리오를 디폴트로 제시하고, Optimistic을 도전 목표로 병행** 제시하는 것이 경영진 신뢰 확보에 효과적.

### 2.3 한국 공공 검증 데이터 인용

> "행안부에 따르면 클라우드 네이티브 전환 후 **장애 발생은 81% 감소**하고  
> **서비스 처리속도는 114% 향상**된 것으로 나타났다.  
> **7년 총소유비용(TCO)도 18.4% 절감** 효과가 확인됐다."  
> — 디지털데일리, 2026.03.11 (행정안전부 발표 데이터 인용)  
> https://www.ddaily.co.kr/page/view/2026031117583162282

**개별 시스템 사례 — 근로복지공단 '고용산재보험토탈서비스':**  
- 응답시간 4.44초 → **0.19초 (23배 단축)**
