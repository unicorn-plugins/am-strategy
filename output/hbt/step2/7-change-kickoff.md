## 2. 영향도 × 지지도 2×2 매트릭스

> 현 시점 기준. Phase 1 파일럿 착수 전 변화관리 주요 목표는 **"설득 대상(Influence↑·Support↓) → 챔피언(↑·↑) 전환"**.

| | **지지도 ↑ (Support High)** | **지지도 ↓ (Support Low)** |
|---|---|---|
| **영향도 ↑ (Influence High)** | ⭐ **챔피언** (강화)<br>① CEO·CFO·CIO<br>③ EA팀<br>⑥ 포털·앱 개발팀 | ⚠️ **설득 대상** (최우선)<br>② 사업본부장<br>⑦ 빌링운영팀<br>⑧ NOC·SOC<br>⑫ 노사협의체 |
| **영향도 ↓ (Influence Low)** | 📣 **홍보대사** (활용)<br>⑤ OSS 개발팀 중 일부<br>⑨ DBA 일부<br>⑩ 보안팀 (Quick Win 수혜) | 👁 **모니터링**<br>④ BSS 개발팀 (혼재)<br>⑬ 외부 파트너<br>⑭ 대리점·상담사<br>⑪ 재무팀 (L2까지는 중립) |

### 2.1 그룹별 전략

| 그룹 위치 | 그룹 | 핵심 전략 |
|--------|----|-------|
| 챔피언 | CEO·CFO·CIO | 월 1회 경영진 대시보드 + 분기 실적 공유 |
| 챔피언 | EA팀 | DDD 퍼실리테이터 역할 공식화 + 이니셔티브 소유권 |
| 챔피언 | 포털·앱 개발팀 | Phase 1 파일럿 주도 (DLR/CSC/MY-APP) — 성공 스토리 생산 |
| ⚠️ 설득 | 사업본부장 | 사업부별 맞춤 Quick Win 제시 (출시 속도 개선 숫자) |
| ⚠️ 설득 | 빌링운영팀 | **연우 담당 1:1 청취** — COBOL 인력 승계·재교육 전제 확약 |
| ⚠️ 설득 | NOC·SOC | 24x7 무중단 설계 공유 + 자동화 교육 기회 제시 |

## 3. 클러스터별 차별화 커뮤니케이션 메시지 (DORA 2025)

> HBT의 클러스터 분포(Phase 3 §2.4): Cluster 1(2) · 2(7) · 3(3) · 4(3) · 5(3) · 6(4) · 7(2).  
> 모든 클러스터에 메시지 차별화.

### 3.1 Cluster 1 (Foundational) — COLL-LGC, FLT-MGT 담당팀
> **핵심 통증**: "생존 모드 — 자원 부족·만성 피로"
>
> *"여러분의 일은 '더 많이 하라'가 아닙니다. AM은 **이 고통을 끝내는 길**입니다.*  
> *FLT-MGT의 Java6·Solaris10은 보안 지적이 반복되는 구조 — 전환 후 보안팀 호출 빈도 50%↓ 목표."*  
> → 구체 약속: Phase 1 내 FLT-MGT Rebuild 착수 + 주 1회 진척 보고

### 3.2 Cluster 2 (Legacy Bottleneck) — CRM·ORD·BILL·STL·DW·SFA·IDC-OPS (7 시스템)
> **핵심 통증**: "불안정 시스템에 끌려다님, 반응형 작업 폭주"
>
> *"AM 후 **반응형 작업이 30~50% 감소**합니다 (DORA 2025 증거 기반).*  
> *여러분의 시간이 가치 창출로 돌아옵니다. 행안부 근로복지공단 사례 — 응답시간 23배 단축.*  
> *우리도 가능합니다."*  
> → 구체 약속: 파일럿 성공 시 야간호출 감소 측정 공개

### 3.3 Cluster 3 (Constrained by Process) — CNTR·CMP·BI-REP 담당팀
> **핵심 통증**: "안정된 시스템 위에서 비효율 프로세스로 트레드밀"
>
> *"AM은 도구가 아닙니다 — **프로세스·거버넌스 재설계**의 기회입니다.*  
> *CAB 1~2주 대기는 STEP 3-5에서 자동 승인 룰로 해소할 계획입니다.*  
> *여러분은 더 이상 '허가를 기다리는 사람'이 아닙니다."*

### 3.4 Cluster 4 (High Impact, Low Cadence) — NMS·PRV·IDP-SSO 담당팀
> **핵심 통증**: "영향력은 크지만 속도가 없어 불균형·번아웃"
>
> *"여러분은 이미 제품 가치에서 인정받고 있습니다. 속도 회복으로 **Cluster 7 진입**이 가능합니다.*  
> *IDP-SSO Replatform은 60+ 앱의 배포 빈도를 올리는 **전제조건** — 핵심 역할입니다."*

### 3.5 Cluster 5 (Stable and Methodical) — PROD·INV·CC-IVR 담당팀
> **핵심 통증**: "안정에 안주 → 속도 부족으로 장기 리스크"
>
> *"현재 안정성을 지키면서 **Throughput만 개선하면 Cluster 7(최상위)** 로 진화할 수 있습니다.*  
> *Replatform의 작은 투자가 큰 효과입니다."*

### 3.6 Cluster 6 (Pragmatic Performers) — DLR·PAY·CSC·MY-APP 담당팀
> **핵심 통증**: "기능적이지만 강한 동기부여 부재"
>
> *"여러분은 Phase 1 파일럿의 **스타**입니다. DLR-PORTAL은 파일럿 1호입니다.*  
> *성공 스토리로 전사에 모멘텀을 만들어 주세요 — 공식 챔피언 역할."*  
> → 구체 약속: 파일럿 성공 시 개발팀 내부 컨퍼런스 세션 배정

### 3.7 Cluster 7 (Harmonious) — REC-AI·CLD-PLAT 담당팀
> *"여러분은 **벤치마크**입니다. 전환 대상이 아닌 To-Be 참조 아키텍처.*  
> *다른 팀의 멘토 역할을 공식화하여 CoP(Community of Practice) 코어로 모십니다."*

## 5. 90분 참여형 워크숍 어젠다

> DORA `07-ai-capabilities-implementation.md` 기반. Phase 0 내에 대상별 최소 1회 실시.

### 5.1 경영진 90분 워크숍 (분기 1회)

```
[0~5분]   오프닝 — 지난 분기 DORA 5 메트릭 실측 공유 (지민)
[5~20분]  이해관계자 맵 리뷰 — 설득 대상 4그룹 현재 상태 (연우)
[20~40분] 3 시나리오 TCO 진척 (재원)
[40~55분] 리스크 Top 5 업데이트 (하늘)
[55~75분] 경영진 의사결정 — 3~5 항목 투표 (CEO 주재)
[75~85분] 다음 분기 스폰서십 액션 확정
[85~90분] 커뮤니케이션 레터 초안 합의
```

### 5.2 개발팀 90분 워크숍 (팀별 1회, 전 9팀)

```
[0~10분]  팀 자가진단 — DORA 7 클러스터 자가평가 (스티키 노트)
[10~30분] VSM 매핑 — code commit → production 화이트보드에 그리기
[30~50분] Event Storming — 도메인 이벤트 과거형 동사로 (핫스팟 스티커)
[50~70분] 우리 팀의 Quick Win 3가지 투표
[70~85분] Phase 1 첫 실험 합의 (시스템·기간·성과 지표)
[85~90분] 다음 워크숍 예약 + 익명 피드백 수집
```

### 5.3 노사협의체 60분 워크숍 (분기 1회)

```
[0~10분]  Phase 진척 공유 (지민·연우)
[10~25분] 인력·역할 변경 사항 투명 공개 (HR팀)
[25~45분] 우려·질문 자유 청취 (연우 퍼실)
[45~55분] 액션 합의 (재교육·배치·예산)
[55~60분] 다음 회의 예약
```
